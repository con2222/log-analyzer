# Log Analyzer Server (TCP-сервер для анализа логов)

Многопоточный TCP-сервер на C++ для анализа текстовых файлов логов.  
Сервер принимает входящие подключения, получает путь к файлу и уровень логирования (INFO, WARNING, ERROR, DEBUG и т.п.), анализирует файл и отправляет клиенту отчет с результатами.

## Возможности

- Работа в режиме TCP-сервера на порту `2222`.
- Асинхронная обработка множества клиентов одновременно (на базе `std::thread`).
- Подсчёт общего числа строк в файле.
- Поиск строк, содержащих заданный уровень логирования.
- Сохранение первых 5 найденных совпадений.
- Отправка готового отчёта напрямую по сети.

## Требования

- ОС на базе Linux / POSIX (используются системные сокеты `<sys/socket.h>`).
- Компилятор с поддержкой C++17 (GCC, Clang).
- CMake (версия 4.2.3 или выше).
- Python 3 (только для генерации тестовых логов).

## Сборка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/con2222/log-analyzer.git
   cd log-analyzer
   ```

2. Создайте директорию для сборки и перейдите в неё:
   ```bash
   mkdir build && cd build
   ```

3. Запустите CMake и соберите проект:
   ```bash
   cmake ..
   cmake --build .
   ```

После сборки исполняемый файл `log-analyzer` появится в папке `build`.

## Генерация тестового файла логов

В репозитории есть скрипт `generator_logs.py`, который создаёт файл `test_logs.txt` с миллионом строк логов. Запустите его:

```bash
python generator_logs.py
```

## Использование

1. **Запустите сервер** (он начнет прослушивать порт 2222):
   ```bash
   ./log-analyzer
   ```
   *Вывод в консоли: `server: waiting for connections...`*

2. **Подключитесь к серверу** из другого терминала. Для этого можно использовать утилиту `nc` (netcat). Отправьте команду в формате `<имя_файла> <уровень_логирования>`:
   ```bash
   echo "test_logs.txt ERROR" | nc localhost 2222
   ```
   *(Также можно запустить `nc localhost 2222` и ввести `test_logs.txt ERROR` вручную).*

Пример ответа от сервера:

```

Первые совпадения:

2. [2023-10-25 10:00:08] [ERROR] Unknown packet received.
3. [2023-10-25 10:00:13] [ERROR] Database timeout occurred.
4. [2023-10-25 10:00:19] [ERROR] Unknown packet received.
```

## Структура проекта

- `LogParser.hpp` / `LogParser.cpp` – класс анализатора. Читает файл и собирает статистику.
- `main.cpp` – точка входа, реализация TCP-сервера, прием подключений и управление потоками (`std::thread`).
- `CMakeLists.txt` – файл сборки.
- `generator_logs.py` – скрипт для генерации тестового файла логов.

## Примечания

- Сервер ожидает, что файл с логами находится в той же директории, откуда был запущен сам сервер (или требует абсолютный путь).
- Программа выводит только первые 5 совпадений, чтобы не перегружать сеть при большом количестве найденных строк.
- Поиск выполняется с учетом регистра (`std::string::find`).
